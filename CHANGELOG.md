# 05/31

## 新系统

* 事件系统
  * 传统的 文字 -\> 选项 -\> 文字 处理系统
  * 当有事件时，玩家界面会被锁在LOGS界面，直到所有事件（依顺序）处理完毕才会回到主界面
  * 处理完毕：当前事件并不处于“等待玩家做选择”且玩家点击界面显示文字将所有文字内容都印出来
  * 事件定义方式纯json（缺点是每个block内容有点多，得定义type）
  * 允许通过定义事件block label来在事件内jump to label或者goto event label
  * 通过事件的选择枝来打断行动
    * 玩家移动途中遇到NPC，事件选择枝可以让玩家原地停下或继续移动
  * 当游戏在更新时，所有事件会被排在更新处理完毕后运行
  * 与玩家无关的事件（例如以NPC为核心触发的事件）在遇到选择枝时会默认选取第一项（无论是否可选）或默认的取消项（如果有设定）
    * 若是与NPC有关，请用branch做自动化的flag管理，只有面向玩家才应当给选择枝。

* 睡眠机制重做
  * 睡眠中的角色会有浮动的睡眠深度，从浅眠到深眠
  * 在睡眠中的角色身边行动，或者对睡眠中的角色互动，会根据当前睡眠深度干扰角色的睡眠，干扰会降低睡眠深度，直到深度归零时角色醒来
    * 浅眠中的角色，睡眠深度会被所有类型的行动干扰
    * 深眠中的角色，睡眠深度只会被痛觉干扰
  * 睡眠深度和时长可以被异常状态修改
    * 喝得烂醉时角色的睡眠深度和时长会增加。
    * 在烂醉下睡觉，角色的睡眠时长会被固定（就算睡到一半醉酒状态消失，已经决定好的睡眠时长也不会变）但睡眠深度会逐渐回归正常水准
  * 角色睡醒时会重新判定所有指令的成功率，且能够取用“刚睡醒”时的指令地文
    * 判定失败的指令会立刻终止
  * 角色醒来后（无论是正常睡醒还是被吵醒）都会检查自身身体状态，然后根据下面几种情况做出派生
    * 眠奸进行中：角色醒来重新判定是否愿意亲密接触，判定成功就转为正常推倒，失败就中止（然后转到【无进行中的互动】派生）
      * 检查到的身体状态会修改上述判定的口上文
    * 无进行中的互动：角色正常醒来，如果检查到了任何异常就添加记忆条目，然后该做啥做啥
  * NPC在睡前会脱衣服，醒来后会穿衣服
    * 穿脱衣物时，若NPC在自己的房间内，会要求房间内的其他人离开房间
    * NPC（应该会）自动接受然后离开
    * 玩家可以选择离开或不离开

# 03/13

## 变更
- 更改地图生成和拼凑机制，现在可以重复使用同一张地图模板，游戏会将每份都独立生成（模板之间暂时没有连接）
- 偏好设置的变更会被保存在Application.dataPath（游戏本地目录\_Data文件夹内），自动保存和读取
- 优化了游戏内更新机制，提高了更新效率。
    - 去掉了部分弱智代码和没有意义的UI更新，上版本耗时5秒的更新现在预计只消耗1秒
    - 降低了部分数据的更新频率
        - 角色记忆，体内物体，阵营道具栏的更新从每分钟1次降低到了每5分钟一次。
        - 非工作岗位无人互动时更新从每分钟1次降低到了每15分钟一次
        - 不与玩家在同一张地图（指没有楼层之间直连通道）的数据更新频率为每6分钟一次。包括以下数据：
            - 行动指令package更新
            - 角色AI寻找工作更新
    - 测试环境下，100个NPC原本耗时20秒的更新，现在预计消耗4秒。还有可优化的空间，但那得先等后续的机制做出来
- 玩家可以加入房间内其他NPC正在执行的家具使用指令（例如休息 娱乐 工作等）

## 已修复BUG
- NPC当前工作的描述会每分钟更新，避免出现已经在执行工作但文字描述不准确的情况
- 玩家单独和与其他NPC一同执行订单制造指令时可以制造出对应道具
- 开始调教行为现在会正确触发对象口上
- 休息指令现在不再被视为娱乐行为，NPC会在体力精力低于50%时（没有工作的话）优先休息
- NPC寻找娱乐行为时不会选择私人房间内的（不会出现NPC没工作的话呆在房间里永远不出来的情况）

### 口上系统改进
- 给大部分指令专用口上预留了位置（部分runtime创造的口上还没填内容，例如餐点指令，种菜指令，监禁指令等）
- 暂时移除了打招呼的口上条目，等找到更好的不会log spam的处理办法后再考虑将其加回来。（保留了每日一次的打招呼）
- 重新整理了logs中口上互动显示条件，现在NPC互相触发的口上内容（例如打招呼）只有当该NPC为玩家选择的目标时才会显示
- 重新整理了logs中口上的显示顺序，同一时间段内 同一角色触发的口上文会被塞到一起（并且追加鼠标悬浮效果）
- 重新分类了口上事件的触发机制

描述事件 "Descriptor"，被玩家看到时触发的一段（参照角色当前状态或行动改变内容的）描述。目前只在与玩家的‘见面事件’中触发

见面事件 "Greeting"，角色进入新的房间时会与新房间内所有角色互相触发的事件。也会在时停解除时触发。目前主要用途为打招呼，如目标为玩家的话额外触发描述事件和导入口上

干扰事件 "Interrupt"，在角色进入新房间时触发，检查房间内的所有行动。也会在角色所处房间内有人执行新行动时触发，检查这个新行动是否会影响/打断角色的当前行动。目前只实装了行动检测，没有实装该检测成立后的效果。
